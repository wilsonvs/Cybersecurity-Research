# Research Methodology

This methodology keeps cybersecurity research notes clear, repeatable, and useful for security operations learning.

## 1. Start With the Event

Capture the basic facts:

- What happened?
- When was it reported?
- Who or what is affected?
- Is exploitation confirmed or theoretical?
- What is the potential impact?

## 2. Verify Sources

Prefer primary and trusted sources:

- Vendor advisories
- CISA alerts
- NIST NVD records
- Official product documentation
- Reputable security research teams
- Incident reports from affected organizations when available

Avoid relying on a single social post or unverified repost.

## 3. Identify Security Relevance

Translate the issue into analyst questions:

- What logs would show this activity?
- What systems are most exposed?
- What user behavior or attacker behavior matters?
- What indicators are useful but not overly broad?
- What controls could prevent, detect, or reduce impact?

## 4. Map to Detection and Response

A useful note should include:

- Possible log sources
- SIEM search ideas
- MITRE ATT&CK techniques when relevant
- Containment actions
- Remediation steps
- Lessons learned

## 5. Keep the Note Practical

Avoid writing only a news summary. The value is in turning threat information into security operations thinking.

A strong research note should help answer: if this happened in an environment, how would I notice, investigate, contain, and explain it?
